Getting Started with datax

datax is a desktop application that helps you deploy and manage data tools across multiple hosts using Docker containers. This guide will help you get started quickly with our unique features including country-specific SCREEN technology and DATAX tools.

What you'll need:

  • ‚úì A computer running Windows, macOS, or Linux
  • ‚úì One or more remote servers with SSH access
  • ‚úì Docker installed on your target servers

Adding Hosts

To deploy applications, you first need to add remote hosts to datax:

  1. 1. Click the "Add Host" button in the main interface
  2. 2. Enter the host details (IP address, SSH port, credentials)
  3. 3. Test the connection to ensure SSH access works
  4. 4. Save the host configuration

Troubleshooting

Common Issues

SSH Connection Failures

If you're unable to connect to a host:

  • ‚Ä¢ Verify SSH service is running on the target host
  • ‚Ä¢ Check firewall rules allow SSH connections
  • ‚Ä¢ Confirm username and credentials are correct
  • ‚Ä¢ Test manual SSH connection from terminal

Docker Deployment Errors

If applications fail to deploy:

  • ‚Ä¢ Ensure Docker is installed and running on target host
  • ‚Ä¢ Verify user has permissions to run Docker commands
  • ‚Ä¢ Check available disk space on host
  • ‚Ä¢ Review application logs for specific error messages

Performance Issues

If experiencing slow performance:

  • ‚Ä¢ Check host resource utilization (CPU, memory, disk)
  • ‚Ä¢ Verify network connectivity between hosts
  • ‚Ä¢ Consider selecting a different SCREEN for your region
  • ‚Ä¢ Review application-specific resource allocation

  • ‚Ä¢ Contact our support team through the Help menu
